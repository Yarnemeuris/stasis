## Design Context

### Users

Stasis is a hardware hackathon platform for Hack Club ‚Äî teens 13‚Äì18 from around the world, culminating in an in-person 4-day build event in Austin, TX. The audience is **dual-cohort and both must be served at once**:

- **Beginners land.** A 13-year-old who has never soldered should feel welcomed, not talked down to. They need clear paths, low-risk entry points (starter projects), and an interface that signals "you belong here."
- **Experienced makers stay.** A teen who already CADs and reflows boards should feel respected ‚Äî no cartoon mascots, no kiddie gamification, no patronizing tone. Depth and craft should reward them.

Context of use: extended sessions on the laptop (picking projects, logging hours, reviewing journals), with mobile check-ins and NFC/inventory usage during the in-person event.

### Brand Personality

**Retrofuturistic single-color console.** Three words: **serialized, instrumented, warm**.

The core reference is a warm monochrome *display surface*: Fallout Pip-Boy, e-ink reader, amber-phosphor CRT, early-80s instrument readout. It is NOT paper, parchment, or manila. The cream isn't cardstock and the dark brown isn't ink ‚Äî they're the lit and unlit states of a single-phosphor screen. The grain on top is a display artifact (subpixel shimmer, phosphor wear), not paper texture.

- *Serialized* ‚Äî things feel like they have a part number. Certificates, manifests, badges, project IDs are rendered as readouts, not rubber-stamps.
- *Instrumented* ‚Äî the interface has dials, readouts, corner brackets, barcode strips. A console you read, not a feed you scroll.
- *Warm* ‚Äî the monochrome is warm, not cold. Cream-on-brown in light mode, orange-on-dark-brown in dark mode. Never cyan, never blue-on-black, never chrome.

The "stasis" name is taken literally: cryo-pod manifests, expedition gear, analog instrument panels, mission patches ‚Äî but displayed on a single-color retro screen, not printed on paper.

### Aesthetic Direction

**Theme split:**
- **Light mode** is the default for the public site and dashboard ‚Äî cream readout (`cream-50/100`) as the lit state, toasted brown (`brown-900`) as foreground, orange (`orange-500`) as the live accent. Think warm e-ink.
- **Dark mode** is a deliberate *room* used on the **starter projects page** (cryo-capsule catalog) and admin pages (back-of-house). It is a single-phosphor display: orange and warm-brown on dark-brown, like a Pip-Boy or amber CRT ‚Äî never cyan-on-black, never pitch black.

**Established design vocabulary** (already shipping, keep using these):
- **Palette:** cream `#F5F3EF`/`#EBE8E0`/`#D5CCB7`, brown `#3B3026`/`#403529`, orange `#D95D39`/`#E86A3A`/`#FC8A58`, yellow `#E1AB55`, taupe `#9C8F88`. Orange is the ONLY saturated accent ‚Äî use it sparingly, let cream/brown carry the weight.
- **Fonts:** Space Grotesk (body/UI), Departure Mono (localFont ‚Äî distinctive display for readouts, labels, serials), Libre Barcode 128 & 39 (actual barcode strips as decoration ‚Äî a core brand motif). Departure Mono is the project's signature display face; it's the thing that makes the site feel like Stasis and not a generic SaaS.
- **Recurring motifs:** `MagneticCorners` (corner brackets that track a target ‚Äî every interactive canvas has these), `NoiseOverlay` / `PlatformNoiseOverlay` (display-surface grain ‚Äî subpixel shimmer / phosphor wear, not paper texture; surfaces are never perfectly clean), `DottedLine` (dashed/dotted rules ‚Äî schematic hints, ruled readouts), `HoverScramble` (text scrambles on hover ‚Äî retro-terminal decoding effect), `ASCIIArt` (monospace art as first-class content), barcode font strips as serialized decoration, status pills that read like inspection readouts.
- **Selection color:** orange-on-cream. The focus ring is orange. Never replace these ‚Äî they're the visual signature.

**Anti-references ‚Äî explicitly NOT:**
- **Generic SaaS / Linear / Vercel dashboard.** No purple-to-blue gradients. No glass cards. No rounded-everything hovering on plain white. No icon-above-heading card grids.
- **Gradient text** (absolute ban ‚Äî it's the #1 AI tell).
- **Side-stripe `border-left: 4px solid accent`** (absolute ban ‚Äî dashboard slop).
- **Generic dark-mode neon** (cyan-on-#0a0a0a). Dark mode here should still feel analog ‚Äî warm-tinted neutrals, not pitch black.

### Hard Rules

- **No rounded corners.** Sharp edges are the default everywhere ‚Äî buttons, cards, inputs, images, modals. Circles are occasionally allowed (avatars, true dots/pills where geometry demands it) but are easy to get wrong, so default to square and only reach for a circle when it's unambiguously the right shape. Never `rounded-md` / `rounded-lg` as a reflex.
- **Departure Mono (the pixel font) needs size discipline.** Because it's a pixel font, mixing sizes produces visible "mixels" ‚Äî pixel grids at different scales that don't align and look broken. Pick as few sizes as the UX allows (ideally one or two across a whole surface) and create hierarchy through other means: weight-where-available, letter-spacing, CAPS vs lowercase, color/value contrast, spacing, rules/dividers, indentation, inversion (light-on-dark blocks). If you find yourself adding a third Departure Mono size, first try to solve the hierarchy problem without one.

### Design Principles

1. **Everything has a part number.** Lean into serialization. Projects, badges, orders, certificates ‚Äî give them IDs, timestamps, stamps, barcode strips. Make the user feel like they're interacting with *issued* artifacts.
2. **Console, not glass, not paper.** Surfaces have display grain (noise overlay) ‚Äî a phosphor/e-ink artifact, not paper texture. Borders are hairline or dashed, never soft-shadowed. Never `backdrop-blur` as decoration. No frosted glass, no drop shadows. When in doubt, imagine the element rendered on a single-color display.
3. **Cream carries, orange accents, brown grounds.** Follow 60-30-10 strictly: cream surfaces, brown text/borders, orange reserved for live state (CTAs, selection, in-progress). When orange appears it should *mean* something.
4. **Instrumented corners, not full frames.** Use `MagneticCorners` and bracket motifs to frame interactive zones instead of boxing everything in cards. Flatten the card nesting; use composition and rhythm instead.
5. **Retro as craft, not costume.** Barcode strips, mono readouts, scramble effects, ASCII art ‚Äî these are allowed because they're functional-looking, not kitsch. No CRT scanlines, no Win95 chrome, no fake-CRT glow. If a retro element doesn't pull its weight, cut it.
6. **Welcome the beginner in the copy, respect the expert in the structure.** Microcopy should be warm, specific, encouraging ("you'll earn your ticket", not "submit entry"). But the information architecture ‚Äî tiers, bits ledger, badge criteria, build logs ‚Äî should be legible and precise enough that an experienced maker trusts it.

### Scope Note

The landing page (`app/page.tsx`) and starter projects page (`app/starter-projects/page.tsx`) are the canonical reference surfaces ‚Äî they have the full attention to detail (magnetic corners, noise, scramble, ASCII, barcode strips, orchestrated motion). Most other user-facing pages (dashboard, project detail, certificate, etc.) share the same design language but haven't reached that bar yet. When designing new features or upgrading existing ones, pull from those two pages as the reference, not from whichever page happens to be nearby.

Admin pages are dark-mode and have a slightly different internal style; they are not user-facing and not a reference for anything.
